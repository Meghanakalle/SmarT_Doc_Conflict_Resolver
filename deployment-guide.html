Deployment Steps

Do Not Modify Your Project

Your React app is already working. Follow these steps without changing your project code.

1

Build Your Project

Open your command line in the project folder and run:

npm run build

This creates a dist folder with optimized production files.

2

Create Redirects File

In the dist folder, create a file named _redirects with this content:

/*    /index.html   200

This ensures client-side routing works correctly.

3

Go to Netlify Drop

Visit https://app.netlify.com/drop

4

Drag and Drop

Drag your entire dist folder to the Netlify Drop zone.

5

Get Your URL

Wait for deployment to complete, then copy your live URL.

Go to Netlify Drop

Project Structure

Your project should look like this. The dist folder is created when you build:

your-project/
├── src/
│ ├── app.tsx
│ ├── main.tsx
│ ├── index.css
│ └── pages/Index.tsx
├── components/
│ └── ui/
│ └── (all shadcn-ui components)
├── dist/ [Generated after build]
│ ├── assets/
│ │ ├── index.[hash].css
│ │ └── index.[hash].js
│ ├── index.html
│ └── _redirects
├── index.html
├── package.json
├── vite.config.ts
└── tailwind.config.js

Note About the dist Folder

The dist folder is automatically created when you run npm run build. You don't need to create it manually.

Troubleshooting

Build Fails

If npm run build fails, check for TypeScript errors in your console and fix them before deploying.

Routing Doesn't Work

Make sure you created the _redirects file in your dist folder with the correct content.

Styles Missing

If styles don't load, check that Tailwind CSS is properly configured in your tailwind.config.js.

Netlify Drop Fails

If dragging the folder doesn't work, try zipping the dist folder first and then dragging the ZIP file.
